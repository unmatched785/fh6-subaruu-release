# v0.6.0-rc.1 스바루런 시작 가이드

## 1. 새 폴더

`v0.6.0-rc.1`은 백그라운드 직접 입력을 시험하는 공개 RC입니다. 안정판 `v0.5.0` 또는 기존 RC 폴더에 덮어쓰지 말고 새 폴더에 압축을 풉니다.

## 2. 버전 확인

프로그램 파일 버전이 `0.6.0-rc.1`인지 확인합니다.

## 3. 기준 환경

- Windows x64
- FH6 한국어 UI
- FHD 1920 x 1080 권장
- 전체 화면 또는 창 전체 화면 권장
- 120 FPS 권장, 60 FPS 검증 완료
- 난이도 옵션의 주행 보조 최대
- `기술`: 꺼짐
- `다음에 할 일`: 꺼짐
- `포커스 상실 시 일시 정지`: 끄기

일반 모드는 게임 창을 계속 보이는 상태로 두고 다른 창으로 전환하지 마세요. 백그라운드 실험 모드만 아래 절차에 따라 브라우저 전환을 허용합니다.

FHD가 기준 권장 해상도입니다. 모든 OCR 판독은 16:9 FHD·QHD·4K 게임 창 캡처를 내부에서 FHD 좌표로 정규화합니다. 울트라와이드와 비 16:9 화면은 지원하지 않습니다.

## 4. 스바루런 준비

- EventLab > `챌린지 플레이하기`
- 공유 코드 `122 697 651`
- 필수 작업차 `1998 SUBARU IMPREZA 22B-STI VERSION`
- 마지막 회차의 두 번째 Enter가 필수 좋아요를 확정하는 안정화 동작인지 확인
- 최근 반복 확인은 회차당 10 SP가 거의 일관됐지만 실제 SP 예산을 확인

챌린지는 반드시 스바루 작업차로 진행합니다. 첫 회차는 사용자가 직접 `1998 SUBARU IMPREZA 22B-STI VERSION`에 탑승해야 합니다. 단독 `스바루런`은 작업차로 챌린지 1회차가 이미 시작된 주행 화면에서 F1을 누릅니다. 테스트·올인원은 작업차를 탄 오픈월드에서 시작하며, 2회차부터는 매크로가 작업차를 다시 선택합니다.

장애물에 강하게 충돌해 속도가 크게 떨어지면 맵의 실패 조건 때문에 챌린지가 실패할 수 있습니다. 이 경우 즉시 `F2`로 정지하고 정상 화면으로 수동 복구한 뒤 다시 시작합니다.

Xbox Game Pass 또는 게임패드 중심 환경에서 공유 코드 입력 뒤 가상 키보드·텍스트 포커스가 남는 문제는 현재 지원 범위 밖의 알려진 한계입니다. 검색으로 넘어가지 않으면 `F2`로 중지하고 수동 복구하세요.

## 5. 백그라운드 실험

1. FH6의 `포커스 상실 시 일시 정지`를 끄고 전체 창 모드로 실행합니다.
2. 첫 차량은 반드시 1998 SUBARU 작업차로 맞춥니다.
3. 이미 챌린지 주행 화면이면 `스바루런`, 오픈월드에서 코드 입력부터 시험하면 `테스트` 탭을 선택합니다.
4. `백그라운드 실험`을 체크합니다. OCR과 위험 탭이 잠겼는지 확인합니다.
5. 테스트 탭은 회차 1, 구매·강화·삭제 0으로 자동 고정됩니다.
6. F1을 누르고 3초 안에 브라우저로 전환해 FH6를 완전히 가립니다.
7. 평소처럼 브라우저의 키보드와 마우스를 사용합니다. FH6는 최소화하지 않습니다.
8. 이상하면 F2로 정지합니다. 완료 후 체크박스는 자동으로 꺼집니다.

챌린지 코드와 W의 백그라운드 수신은 이 RC의 핵심 미검증 항목입니다. 처음에는 테스트 런 1회로 확인하세요.

## 6. FH6 398.92.0 패치 경고

- 22B 1대: `313,500 CR` / `30 SP` / 슈퍼 휠스핀 1회
- 스바루 프리셋 33대: `10,345,500 CR` / `990 SP` / 슈퍼 휠스핀 33회
- 구 EventLab 이벤트: 사용자 실측상 `50,000`점 = 최대 `1 SP`, 신 버전에서 폐기
- 신 챌린지 코드 `122 697 651`: 사용 전 1회 보상 수동 검증 필수
- 기본 111회: 9~10 SP 관측 기준 `999~1,110 SP`, 재로딩과 선택형 결과 OCR 대기를 포함해 약 93분 이상

스바루 프리셋 33대에는 990 SP가 필요합니다. 기본값이 99회가 아닌 111회인 이유는 `111 × 9 = 999 SP`가 되어 과거 9 SP 관측을 적용해도 필요량을 넘기기 때문입니다. 최근에는 10 SP가 거의 일관되게 확인됐지만 구매 전 실제 SP 예산 검증을 최종 기준으로 둡니다.

기본 흐름은 `W`를 계속 누른 채 25초 주행하고 챌린지 성공으로 가정합니다. 중간 회차는 Esc로 재시작하고 25초 로딩을 기다리며, 마지막 회차는 Enter로 종료한 뒤 5초 후 좋아요 Enter를 누릅니다. `결과 OCR 사용`을 켜면 성공·실패를 판독해 키를 선택하며, 불명확하면 안전 정지합니다.

## 7. 먼저 알아둘 키

- `F1`: 실행
- `F2`: 중지 요청
- `F3`: 중지 요청 후 종료
- `F4`: 휠스핀 진단 캡처

## 8. 첫 시험

1. 오픈월드에서 `테스트` 탭을 엽니다.
2. `챌린지만` 프리셋을 선택합니다. 기본값은 `런 1 / 구매 0 / 강화 0 / 삭제 0`입니다.
3. F1을 누르고 코드 검색, W 유지, 성공 고정 또는 선택형 OCR 키 선택, 마지막 종료와 좋아요 Enter, 오픈월드 복귀를 직접 봅니다.
4. 시작 전후 SP를 확인합니다. 최근에는 회차당 10 SP가 거의 일관되게 확인됐지만 실제 SP 예산을 최종 기준으로 둡니다.
5. 이 구간이 성공한 뒤에만 `연결검증`으로 `런 1 / 구매 1 / 강화 1 / 삭제 0`을 시험합니다.
6. 삭제·자동 판매는 마지막에 각각 1회만 별도로 시험합니다.

이 프로그램의 이미지/OCR 판정은 제한적인 추정입니다. 원하는 차량의 구매·삭제·판매를 보증하지 않습니다.

## 9. 문제가 생기면

`F2`로 멈추고 `log/last-run-status.txt`와 가장 최근 `session-*.log`를 확인합니다. 로그나 PNG를 공유하기 전 Windows 사용자 이름, 로컬 경로, 게이머태그, CR 등 계정 정보를 지우거나 가리세요.

상세 설명은 `README.md`와 `GUIDE.md`에 있습니다.
