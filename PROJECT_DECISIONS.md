# FH6 Subaruu 확정 결정

이 문서는 후속 수정에서 이미 해결된 동작을 실수로 되돌리지 않기 위한 기준입니다. 아래 항목을 바꾸려면 실제 게임 검증 결과와 변경 이유를 함께 기록해야 합니다.

## 되돌리면 안 되는 동작

1. 구 EventLab 이벤트 자동 경로는 레거시로만 보존합니다. 새 스바루런·테스트·올인원에서는 호출하지 않습니다.
2. `스바루작` 구매·마스터리 프리셋은 종료된 레거시입니다. 기본값이나 사용자 버튼으로 다시 노출하지 않습니다.
3. 새 챌린지는 반드시 `1998 SUBARU Impreza 22B-STI Version` 작업차로 진행합니다.
4. 테스트·올인원 다회차는 2회차부터 챌린지 진입 전에 스바루 작업차를 자동 재선택합니다. 구매·마스터리 차량을 탄 채 다음 챌린지를 시작하면 안 됩니다.
5. 테스트와 올인원의 기본 챌린지 코드는 모두 `122 697 651`입니다. 두 설정이 서로 다른 옛 코드로 되돌아가면 안 됩니다.
6. 기본 결과 대기 시작은 25초입니다. 최근 반복 확인에서 회차당 10 SP가 거의 일관됐고 60 FPS도 검증한 값입니다.
7. `결과 OCR 사용`은 선택 옵션이며 기본값은 꺼짐입니다. 꺼져 있으면 챌린지 성공으로 가정하여 중간 회차는 `Esc`로 재시작하고 마지막 회차는 `Enter`로 종료합니다. OCR을 켜면 성공·실패 결과를 판독해 키를 선택합니다.
8. 휠스핀 탭의 기본 종류는 `슈퍼`입니다.
9. 모든 OCR은 16:9 입력을 FHD 좌표로 정규화합니다. FHD를 권장하고 QHD·4K도 지원하지만 울트라와이드와 비 16:9 화면은 지원하지 않습니다.
10. 슈퍼 휠스핀의 중복차 팝업은 빈 프레임 없이 다음 차량 팝업으로 바로 바뀔 수 있습니다. 팝업 화면 변화량으로 다음 차량을 다시 처리하는 보강을 유지합니다.
11. 장시간 실행과 자리를 비운 무인 이용은 권장하지 않습니다. 새 맵·게임 패치·화면 상태는 짧은 테스트런으로 먼저 확인합니다.
12. 기본 런 횟수는 99회가 아니라 111회입니다. 스바루 프리셋 33대에 필요한 990 SP를 기준으로 과거 관측 9 SP를 전 회차에 적용해도 `111 × 9 = 999 SP`가 되도록 여유를 둡니다. 최근에는 10 SP가 거의 일관됐지만 올인원의 실제 SP 예산 검증을 최종 기준으로 사용합니다.
13. 마지막 좋아요 Enter는 사용자 실검증이 끝난 안정화 동작입니다. 근거 없이 제거하거나 대기값을 되돌리지 않습니다.
14. 휠스핀의 `1989 NISSAN S-CARGO`, `2013 WULING`, `2020 WULING`은 모두 FE 후보로 보수적으로 보호합니다. 실제 S-Cargo를 OCR이 WULING으로 오인해도 보라색 카드의 FE 표식을 놓쳤다는 이유로 판매하면 안 됩니다.
15. 정식 v0.5.0 권장 환경은 한국어 UI, FHD 16:9, 전체 화면 또는 창 전체 화면, 주행 보조 최대, 기술 HUD 꺼짐, 다음에 할 일 꺼짐, 포커스 상실 시 일시 정지 끄기입니다. 120 FPS를 권장하고 60 FPS도 검증했습니다.
16. 백그라운드 실험 모드는 브라우저가 FH6를 완전히 가린 상태를 목표로 하지만 FH6 최소화는 지원하지 않습니다.
17. 백그라운드 실험 모드에서는 스바루런과 테스트-챌린지만 허용합니다. 결과 OCR을 끄고 테스트 회차 1, 구매·강화·삭제 0을 강제하며 구매·마스터리·삭제·휠스핀·올인원을 잠급니다.
18. 백그라운드 실행 중 화면 캡처, OCR, 실제 마우스 클릭, FH6 전경 전환이 호출되면 계속 진행하지 않고 실패 폐쇄 방식으로 중지합니다.
19. 같은 챌린지 안의 반복 런은 작업차 재선택이 필요하지 않으므로 백그라운드 모드에서도 허용합니다. 두 번째 테스트/올인원 회차는 허용하지 않습니다.
20. 백그라운드 챌린지 코드는 숫자와 검색 Enter를 FH6 창으로 직접 보내며 성공 화면을 검증했다고 주장하지 않습니다. 실게임 검증 전까지 RC 기능으로 유지합니다.

## 현재 지원 범위 밖의 알려진 한계

- Xbox Game Pass 또는 게임패드 중심 환경에서 공유 코드 입력 시 가상 키보드/텍스트 포커스가 남아 검색으로 넘어가지 않는 문제가 있습니다. v0.5.0 승격을 막지는 않지만 현재 자동 진행 지원 범위에는 포함하지 않습니다.
- 이 문제를 이유로 작업차 재선택, 새 맵 코드, OCR 선택 옵션을 제거하거나 구 EventLab 경로로 되돌리지 않습니다.
- 임의의 고정 화면 좌표 클릭은 아직 확정 해결책이 아닙니다. `GAMEPASS_CODE_INPUT_DESIGN.md`의 검증 절차를 거쳐야 합니다.
