# FH6 스바루런 보조 도구 v0.6.0-rc.1

## ⚠️ 반드시 스바루 작업차를 타고 시작하세요

> **필수 차량: `1998 SUBARU IMPREZA 22B-STI VERSION`**
>
> 다른 차량으로는 챌린지 기술 포인트 10점이 정상 적립되지 않습니다. 첫 회차는 차량을 확인하지 않으므로 사용자가 반드시 작업차에 직접 탑승한 뒤 실행해야 합니다.

Forza Horizon 6에서 반복 키 입력, 차량 구매·마스터리·삭제 흐름, 일반·슈퍼 휠스핀 처리를 보조하는 비공식 Windows x64 도구입니다.

`v0.6.0-rc.1`은 브라우저가 FH6를 완전히 가린 상태에서 스바루런과 챌린지 코드 입력을 직접 창 메시지로 보내는 백그라운드 실험 기능을 추가한 공개 RC입니다. 안정판은 `v0.5.0`이며, `v0.4.3-final-20260713`은 구 EventLab 이벤트 경로를 보존한 레거시입니다.

## FH6 398.92.0 이후 스바루작 상태

2026년 7월 14일 KST에 배포된 FH6 398.92.0은 1998 Subaru Impreza 22B-STi 가격과 스킬 포인트 파밍 규칙을 바꿨습니다. 아래 계산은 패치 후 5% 할인 적용가 `313,500 CR`을 기준으로 합니다. 할인 전 표시는 `330,000 CR`입니다.

| 항목 | 패치 후 기준 |
| --- | ---: |
| 22B 1대 | `313,500 CR` |
| 22B 1대 마스터리 | `30 SP` → 슈퍼 휠스핀 1회 |
| 스바루 프리셋 33대 | `10,345,500 CR` + `990 SP` → 슈퍼 휠스핀 33회 |
| 현금 손익분기 | 슈퍼 휠스핀 1회당 평균 `313,500 CR` |

손익분기는 보상 차량·의상·경적의 수집 가치와 남겨 둔 22B의 가치를 CR로 환산하지 않은 값입니다. 프로그램의 기본 삭제 흐름으로 차량을 제거하면 그 차량 가격은 회수되지 않습니다.

[Steam 398.92.0 패치노트](https://store.steampowered.com/news/app/2483190/view/689762798147733105)는 22B 가격 조정을, [Forza 공식 Series 3 안내](https://forza.net/news/forza-horizon-6-series-3)는 파밍 익스플로잇 수정을 명시하지만 수치까지 공개하지 않았습니다. 패치 뒤 기존 EventLab 이벤트는 유효 점수가 `50,000`점, 즉 완료 1회당 최대 `1 SP`로 제한됐다는 [사용자 실측](https://www.reddit.com/r/ForzaHorizon/comments/1uvjqp6/eventlab_skill_points_capped/)이 있어 v0.5.0에서는 해당 경로를 사용하지 않습니다.

새 로컬 경로는 EventLab의 `챌린지 플레이하기`와 공유 코드 `122 697 651`을 사용합니다. 최근 반복 확인에서는 회차당 10 SP가 거의 일관되게 확인됐고, 과거에는 9 SP 관측도 있었습니다. 기본값은 99회가 아니라 111회입니다. 스바루 프리셋 33대에 필요한 990 SP를 기준으로 `111 × 9 = 999 SP`, `111 × 10 = 1,110 SP`가 되어 관측 범위 안에서 여유를 남깁니다. 25초 주행·재로딩 25초·좋아요 대기 5초를 사용하며, 실제 구매 전 프로그램의 SP 예산 검증을 최종 기준으로 둡니다. `결과 OCR 사용`은 기본 꺼짐이며, 꺼져 있으면 성공으로 가정해 중간 회차는 `Esc`, 마지막 회차는 `Enter`를 누릅니다. 옵션을 켜면 실패 화면의 `Enter 재시도 / Esc 종료`와 성공 화면의 `Esc 재시도 / Enter 계속`을 OCR로 구분하며, 판정이 불명확하면 안전 정지합니다. 좋아요 확정 단계는 실게임 검증이 끝난 안정화 동작이므로 유지합니다.

모든 OCR 판독은 16:9 화면을 FHD 좌표로 처리합니다. FHD·QHD·4K 게임 창 캡처는 내부에서 FHD로 정규화하지만 울트라와이드와 비 16:9 화면은 지원하지 않습니다.

프로그램은 보유 CR과 실제 구매·마스터리 성공을 확정적으로 읽지 않습니다. 처음에는 `챌린지 1 / 구매 1 / 마스터리 1 / 삭제 0`으로 전 과정을 직접 확인하세요.

`v0.5.0`의 테스트·올인원은 챌린지 종료 뒤 자동차 메뉴의 현재 SP를 두 번 OCR로 확인하고, `현재 SP ≥ 강화 대수 × 프리셋별 SP`일 때만 구매 단계로 진행합니다. 기준은 마즈다 21 SP/대, 라부엘토 39 SP/대, 레거시 스바루 30 SP/대입니다. 이 검증은 16:9 FHD·QHD·4K 한국어 UI를 지원하며, 비 16:9 화면·판독 실패·불일치·부족은 모두 안전 정지합니다.

구매·마스터리·삭제의 새 기본 프리셋은 `마즈다작` 47대입니다. 구형 `스바루작` 구매·마스터리 경로는 기존 설정 호환을 위한 레거시 코드로만 남아 있으며 기본값이나 버튼으로 제공하지 않습니다.

## 먼저 알아야 할 한계

이 프로그램은 FH6의 공식 API를 쓰지 않습니다. 정해진 위치와 시간에 키·마우스 입력을 보내고, 일부 화면을 이미지 비교나 OCR로 추정합니다.

화면 판정이 성공했다고 해서 아래 사실이 증명되는 것은 아닙니다.

- 원하는 차량을 실제로 구매했는지
- 현재 카드가 이번 실행에서 산 차량인지
- 삭제·판매 대상이 사용자가 의도한 차량인지
- 게임 업데이트 뒤에도 메뉴 순서와 입력 타이밍이 같은지

프로그램의 판정과 중단 조건은 제한적인 보조 신호입니다. 삭제·중복차 판매의 정확성을 보증하는 보호 장치로 취급하지 마세요. 삭제와 판매는 되돌리기 어렵습니다.

게임 업데이트, 매주 목요일 23:30 전후의 시즌 전환, DLC·차고·즐겨찾기 상태, 해상도, 로딩 속도, 포커스 이동에 따라 입력이 어긋날 수 있습니다. 처음 쓰는 PC나 환경이 바뀐 날에는 삭제와 판매가 없는 작은 횟수부터 직접 확인하세요.

## 백그라운드 실험 모드

`v0.6.0-rc.1`의 목표는 FH6가 메인 모니터에서 브라우저에 완전히 가려져도 사용자가 브라우저의 키보드와 마우스를 평범하게 사용하면서 FH6에만 키 입력을 전달하는 것입니다. 아직 실게임 검증 전인 RC 기능입니다.

- FH6 `HUD 및 게임플레이 > 포커스 상실 시 일시 정지`를 끕니다.
- 전체 창 모드 또는 창 모드를 사용하고 FH6를 최소화하지 않습니다.
- `백그라운드 실험`을 체크하면 결과 OCR이 꺼지고 구매·마스터리·삭제·휠스핀·올인원 탭이 잠깁니다.
- 허용 범위는 이미 챌린지 안에서 시작하는 `스바루런`과 오픈월드에서 시작하는 `테스트-챌린지만`입니다.
- 테스트 회차는 1, 구매·강화·삭제는 0으로 강제됩니다. 같은 챌린지 안의 스바루런 반복 횟수는 유지됩니다.
- F1을 누르면 3초 준비 시간이 생깁니다. 그 안에 브라우저로 전환합니다.
- 챌린지 코드 숫자, 검색 Enter, 메뉴 이동과 W는 FH6 창으로 직접 전송합니다. 코드 입력 성공 여부는 아직 화면으로 검증하지 않습니다.
- 실행 중 OCR·화면 캡처·마우스 클릭·전경 전환이 호출되거나 FH6가 최소화되면 안전 정지합니다.
- 실행이 끝나거나 실패하면 W를 해제하고 체크박스를 자동으로 끕니다.

브라우저의 F1·F2·F3도 프로그램의 전역 시작·중지·종료 키로 감지됩니다. 실행 중에는 F2가 긴급정지라는 점을 기억하세요. Xbox Game Pass/게임패드 가상 키보드 환경의 코드 입력 문제는 이 RC에서도 미검증입니다.

## 다운로드와 무결성 확인

백그라운드 기능 시험판은 `v0.6.0-rc.1`, 안정판은 `v0.5.0`입니다. 구 EventLab 경로가 필요한 경우에만 레거시 `v0.4.3-final-20260713`을 사용하세요.

시험 사용자는 GitHub Releases의 `v0.6.0-rc.1`, 안정판 사용자는 `v0.5.0`에서 `subaruu.zip`과 `subaruu.zip.sha256`을 함께 받으세요.

1. ZIP과 함께 게시된 `subaruu.zip.sha256`을 받습니다.
2. PowerShell에서 `Get-FileHash .\subaruu.zip -Algorithm SHA256`을 실행합니다.
3. 두 해시가 완전히 같은지 확인합니다.
4. ZIP 안의 `BUILD_INFO.txt`에서 버전·소스 커밋·빌드 정보를 확인합니다.

RC 해시는 릴리즈 생성 뒤 `RC_RELEASE_v0.6.0-rc.1.md`에 고정합니다. `FINAL_RELEASE_v0.5.0.md`는 안정판 기록이며 덮어쓰지 않습니다.

## 설치와 업그레이드

**기존 설치 폴더에 덮어쓰지 마세요.**

1. 새 빈 폴더를 만듭니다.
2. `subaruu.zip`을 새 폴더에 압축 해제합니다.
3. `subaruu/subaruu.exe`를 실행합니다.
4. 이전 설정이 정말 필요할 때만 기존 `config/subaruu.conf`의 값을 새 설정 파일에 수동으로 옮깁니다.

구버전의 실행 파일, 이미지 템플릿, OCR 자료, 설정을 섞으면 어떤 자산이 실제로 사용됐는지 알 수 없습니다. 폴더 전체 복사나 덮어쓰기는 지원하지 않습니다.

Windows x64 self-contained single-file 빌드이므로 별도 .NET Desktop Runtime 설치는 필요하지 않습니다. `x64`, `data`, `config`, `licenses` 폴더는 `subaruu.exe`와 함께 두세요.

OCR 네이티브 DLL은 Microsoft Visual C++ 런타임을 사용합니다. 실행 또는 OCR 초기화 때 `VCRUNTIME`/`MSVCP` DLL 오류가 나면 Microsoft의 최신 Visual C++ v14 Redistributable x64를 설치하세요: https://aka.ms/vc14/vc_redist.x64.exe

주요 파일:

- `subaruu.exe`: 프로그램
- `BUILD_INFO.txt`: 버전·소스·빌드 정보
- `SHA256SUMS.txt`: ZIP 내부 파일 해시
- `config/subaruu.default.conf`: 기본 설정
- `data/eventlab-templates/`: 화면 비교 자료
- `data/tessdata/`: OCR 언어 자료
- `x64/`: Tesseract·Leptonica 네이티브 파일
- `licenses/`: 오픈소스·글꼴 라이선스

## 키

- `F1`: 현재 선택된 탭 실행
- `F2`: 실행 중지 요청
- `F3`: 실행 중지 요청 후 프로그램 종료
- `F4`: 휠스핀 진단 캡처

`F1`은 확인창 없이 실행됩니다. `F2`와 `F3`도 게임이 이미 받은 입력이나 이미 확정된 삭제·판매를 되돌리지는 못합니다.

## 첫 실행 준비

권장·검증 환경:

- Windows x64
- FH6 한국어 UI
- FHD 1920 x 1080 권장
- 전체 화면 또는 창 전체 화면 권장
- 120 FPS 권장, 60 FPS 검증 완료
- 난이도 옵션의 주행 보조를 최대로 설정
- `HUD 및 게임플레이 > 기술`: 꺼짐
- `HUD 및 게임플레이 > 다음에 할 일`: 꺼짐
- `HUD 및 게임플레이 > 포커스 상실 시 일시 정지`: 끄기

FHD가 기준 권장 해상도입니다. QHD·4K도 16:9이면 내부에서 FHD 좌표로 정규화하지만, 울트라와이드와 비 16:9 화면은 지원하지 않습니다. 일반 모드는 실행 중 다른 창으로 전환하면 안 됩니다. 백그라운드 실험 모드만 브라우저 전환을 허용하며 FH6 최소화는 여전히 금지합니다.

스바루런 준비:

- EventLab `챌린지 플레이하기` 공유 코드: `122 697 651`
- 필수 작업차: `1998 SUBARU IMPREZA 22B-STI VERSION`
- 마지막 회차의 두 번째 Enter가 필수 좋아요 화면을 확정하는 안정화 동작인지 확인
- 최근 반복 확인은 회차당 10 SP가 거의 일관됐지만 실제 SP 예산 확인

챌린지는 반드시 스바루 작업차에 탑승한 상태로 진행해야 합니다. 단독 `스바루런` 탭은 차량 변경이나 코드 검색을 하지 않으므로, 작업차로 챌린지 1회차가 이미 시작되어 달릴 수 있는 화면에서 F1을 누르세요.

첫 회차의 차량 OCR과 자동 재탑승은 사용하지 않습니다. `테스트`와 `올인원`도 시작 전에 사용자가 직접 `1998 SUBARU IMPREZA 22B-STI VERSION`에 탑승해야 합니다. 다회차 실행은 2회차부터 구매·마스터리 차량 대신 작업차를 다시 선택한 뒤 챌린지에 진입합니다.

주행 중 장애물에 강하게 충돌해 속도가 크게 떨어지면 챌린지 자체가 실패할 수 있습니다. 이는 입력 순서가 아니라 맵의 주행·실패 조건에 따른 제한이며, 실패 화면에서는 이후 입력 순서가 맞지 않게 됩니다. 즉시 `F2`로 정지하고 정상 화면으로 수동 복구한 뒤 다시 시작하세요.

Xbox Game Pass 또는 게임패드 중심 환경에서 공유 코드 입력 뒤 가상 키보드·텍스트 포커스가 남는 문제는 현재 지원 범위 밖의 알려진 한계입니다. 검색으로 넘어가지 않으면 `F2`로 중지하고 수동 복구하세요. 빈 코드나 즐겨찾기 우회도 v0.5.0 신 경로에서 지원하지 않습니다.

처음 실행할 때는 다음 순서를 권합니다.

1. 프로그램만 열고 설정과 탭을 확인합니다.
2. 삭제 횟수를 0으로 둔 작은 테스트를 관찰합니다.
3. 구매·마스터리·휠스핀을 각각 작은 횟수로 확인합니다.
4. 삭제와 중복차 판매는 대상 카드를 사람이 직접 확인한 뒤 최소 횟수로 시험합니다.
5. 결과가 본인 환경에서 일관될 때만 횟수를 늘립니다.

테스트런도 실제 키 입력을 보내는 자동화입니다. 무해한 시뮬레이션이 아닙니다.

| 스바루런 | 챌린지 1회차 주행 화면 | 즉시 `W`를 유지하고 25초 주행. 기본은 성공 고정으로 중간 Esc·마지막 Enter이며, 선택적으로 결과 OCR을 사용한 뒤 좋아요 Enter |
| 구매 | 자동차 컬렉션의 구매 대상 차량 카드 | 설정된 입력 순서로 구매 반복 |
| 마스터리 | 페스티벌 > 경매장 > 경매 시작 > 차량 선택 목록 | 최근 추가됨 기준 탑승·마스터리 입력 |
| 삭제 | 페스티벌 > 경매장 > 경매 시작 > 차량 선택 목록 | 최근 추가됨 기준 첫 칸 삭제 반복 |
| 테스트 | 작업차를 탄 오픈월드 | 챌린지부터 짧은 횟수로 올인원과 같은 경로 실행 |
| 올인원 | 작업차를 탄 오픈월드 | 챌린지·주행·구매·마스터리·삭제 연결 실행 |
| 휠스핀 | 나의 HORIZON의 일반 또는 슈퍼 휠스핀 카드 | 휠스핀과 중복차 처리 |

구매 메뉴의 위치와 최근 추가됨 정렬은 계정마다 달라질 수 있습니다. 작업 중 다른 차량을 받거나 휠스핀을 직접 열면 정렬이 바뀔 수 있습니다.

프리셋은 입력 횟수와 이동값을 채워 주는 기능입니다. 현재 게임 상태가 맞다는 뜻도, 실행을 승인한다는 뜻도 아닙니다.

## 휠스핀

일반·슈퍼 휠스핀은 독립 실행입니다. 테스트와 올인원에는 연결되어 있지 않습니다.

시작 위치:

```text
나의 HORIZON > 실행할 일반/슈퍼 휠스핀 카드에 포커스
```

주요 옵션:

- `레전더리 보관`: 기본 켜짐
- `에픽 보관`: 기본 꺼짐
- `의상,경적 올클`: 실제로 모두 보유한 계정에서만 사용
- `상세 화면 저장`: 기본 꺼짐

독점차·Forza Edition·중복차 판정은 내장 목록과 이미지/OCR 추정에 의존합니다. 인식이 맞지 않으면 잘못 보관하거나 판매할 수 있습니다. 가치가 큰 차량이 걸린 계정에서는 자동 판매를 사용하지 않는 것이 가장 확실합니다.

완료 뒤 보상 장부가 열리고 아래 파일이 `result/wheelspin-날짜-시간-종류/`에 저장됩니다.

- `wheelspin-summary.rtf`
- `wheelspin-result.rtf`
- `wheelspin-result.txt`

휠스핀은 목표 횟수 완료 후 휠 메뉴에 머무릅니다.

## 로컬 데이터와 개인정보

프로그램은 설정·로그·결과·필요한 진단 캡처를 실행 파일 옆에 저장합니다. 현재 소스에는 이를 인터넷으로 전송하는 기능이 없습니다.

- `config/subaruu.conf`: 사용자 설정
- `log/session-*.log`: 탭·횟수·진행·오류·로컬 파일 경로
- `log/last-run-status.txt`: 마지막 실행 상태
- `log/*.png`, `log/capture/*.png`: 오류·수동·선택적 상세 캡처
- `result/`: 휠스핀 보상 기록

캡처에는 FH6 화면의 게이머태그, CR, 차량, 알림 등 계정 정보가 보일 수 있고, 로그에는 Windows 사용자 이름이 포함된 로컬 경로가 남을 수 있습니다.

도움을 요청할 때 폴더 전체를 그대로 올리지 마세요. 가장 최근의 관련 로그만 복사하고, 사용자 이름·로컬 경로·게이머태그 등 식별 가능한 내용을 지우세요. PNG는 확인 후 필요한 부분만 잘라내거나 가린 뒤 공유하세요. 비밀번호나 인증 토큰을 요구하는 지원 절차는 없습니다.

자세한 개인정보 안내는 배포 ZIP 안의 `PRIVACY.md`를 참조하세요.

## 문제 확인 순서

1. 즉시 `F2`를 누릅니다.
2. 게임 화면을 정상 시작 위치로 되돌립니다.
3. 가장 최근 `log/last-run-status.txt`와 `session-*.log`를 확인합니다.
4. 횟수를 1로 줄이고 삭제·판매를 끈 상태에서 같은 구간을 관찰합니다.
5. 입력이 빠르면 해당 탭의 대기·간격 값을 늘립니다.
6. 게임 업데이트나 시즌 전환 뒤라면 제조사·차량 위치를 다시 셉니다.

설정 파일이 깨졌다면 프로그램을 종료하고 `config/subaruu.conf`를 별도 보관한 뒤 제거하세요. 다음 실행에서 기본 설정을 다시 만듭니다.
