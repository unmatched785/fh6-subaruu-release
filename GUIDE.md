# FH6 스바루런 상세 사용 가이드

이 문서는 새 챌린지 경로와 선택형 결과 OCR을 정식 기준으로 승격한 `v0.5.0` 사용 가이드입니다. 빠른 시작은 `STARTER_GUIDE.md`, 설치·개인정보 안내는 `README.md`를 먼저 읽으세요. `v0.4.3-final-20260713`은 구 EventLab 이벤트 경로를 보존한 레거시이며 이 흐름을 포함하지 않습니다.

## 1. 동작 방식

이 도구는 게임 상태를 공식 데이터로 읽지 않습니다. 창 화면 일부를 캡처해 이미지 비교·OCR을 시도하고, 정해진 순서와 대기 시간으로 키·마우스 입력을 보냅니다.

이미지 일치 점수나 OCR 결과는 화면에 대한 추정입니다. 원하는 차량의 소유권, 구매 성공, 삭제·판매 대상의 정확성을 증명하지 못합니다. 사람이 화면을 확인하지 않아도 되는 무인 자동화로 사용하지 마세요.

특히 다음 작업은 되돌리기 어렵습니다.

- 차량 삭제
- 휠스핀 중복 차량 판매
- 잘못된 차량 구매
- 잘못된 차량의 마스터리 포인트 사용

## 2. 새 폴더에서 시작

버전을 섞지 마세요.

1. 릴리즈 ZIP의 SHA-256을 확인합니다.
2. 새 빈 폴더에 압축을 풉니다.
3. 기존 실행 파일·`data`·`x64`·템플릿을 복사하지 않습니다.
4. 이전 설정이 필요하면 새 버전을 한 번 실행한 뒤, 필요한 값만 기존 `config/subaruu.conf`에서 수동으로 옮깁니다.

기존 설치 폴더에 덮어쓰면 오래된 이미지 자료나 OCR 파일이 남을 수 있어 지원 대상에서 제외합니다.

별도 .NET Desktop Runtime은 필요하지 않습니다. 다만 OCR 네이티브 DLL 초기화에서 `VCRUNTIME` 또는 `MSVCP` 파일 누락 오류가 나면 Microsoft의 최신 Visual C++ v14 Redistributable x64를 설치합니다: https://aka.ms/vc14/vc_redist.x64.exe

## 3. 게임 환경

기준 환경:

| 항목 | 값 |
| --- | --- |
| 운영체제 | Windows x64 |
| 게임 UI | 한국어 |
| 해상도 | FHD 1920 x 1080 권장 |
| 화면 모드 | 전체 화면 또는 창 전체 화면 권장 |
| 프레임 제한 | 120 FPS 권장, 60 FPS 검증 완료 |
| 난이도 | 주행 보조 최대 |
| HUD 및 게임플레이 > 기술 | 꺼짐 |
| HUD 및 게임플레이 > 다음에 할 일 | 꺼짐 |
| HUD 및 게임플레이 > 포커스 상실 시 일시 정지 | 끄기 |

FHD가 기준 권장 해상도입니다. QHD·4K도 16:9이면 내부에서 FHD 좌표로 정규화하지만, 다른 언어·UI 배율·울트라와이드·비 16:9 화면은 지원하지 않습니다. 일반 창 모드도 검증 기준이 아닙니다.

`포커스 상실 시 일시 정지`를 끄면 게임 창이 잠깐 포커스를 잃었을 때 FH6가 자동으로 멈추는 것을 방지합니다. 다만 일부 단계는 전경 키보드·마우스 입력을 사용하므로 실행 중 다른 창으로 전환해도 된다는 뜻은 아닙니다.

스바루런 준비:

- EventLab > `챌린지 플레이하기`
- 공유 코드 `122 697 651`
- 필수 작업차 `1998 SUBARU IMPREZA 22B-STI VERSION`
- 마지막 회차의 두 번째 Enter가 필수 좋아요 화면을 확정하는 안정화 동작인지 확인
- 최근 반복 확인은 회차당 10 SP가 거의 일관됐지만 실제 SP 예산을 확인

챌린지는 반드시 스바루 작업차에 탑승한 상태로 진행해야 합니다. 단독 `스바루런` 탭은 차량 변경·챌린지 검색·최초 로딩을 수행하지 않으므로, 작업차로 챌린지 1회차가 이미 시작된 주행 화면에서 실행합니다.

첫 회차의 차량 OCR과 자동 재탑승은 사용하지 않습니다. `테스트`와 `올인원`도 사용자가 직접 `1998 SUBARU IMPREZA 22B-STI VERSION`에 탑승한 오픈월드에서 시작해야 합니다. 다회차에서는 2회차부터 작업차를 이미지로 다시 선택한 뒤 챌린지 진입을 진행합니다.

주행 중 장애물에 강하게 충돌해 속도가 크게 떨어지면 맵의 실패 조건 때문에 챌린지가 종료될 수 있습니다. 실패 화면에서는 정상 결과 화면을 전제로 한 이후 입력이 어긋나므로 즉시 `F2`로 정지하고 수동 복구 후 다시 시작합니다.

Xbox Game Pass 또는 게임패드 중심 환경에서 공유 코드 입력 뒤 가상 키보드·텍스트 포커스가 남는 문제는 현재 지원 범위 밖의 알려진 한계입니다. 검색으로 넘어가지 않으면 `F2`로 중지하고 수동 복구하세요. v0.5.0은 빈 코드나 즐겨찾기 우회도 지원하지 않습니다.

### FH6 398.92.0 이후 효율

2026년 7월 14일 KST 기준 22B의 5% 할인 적용가는 `313,500 CR`입니다. 할인 전 표시는 `330,000 CR`입니다.

| 단위 | 필요한 CR | 필요한 SP | 받는 보상 |
| --- | ---: | ---: | ---: |
| 22B 1대 | `313,500` | `30` | 슈퍼 휠스핀 1회 |
| 스바루 프리셋 33대 | `10,345,500` | `990` | 슈퍼 휠스핀 33회 |

따라서 슈퍼 휠스핀의 현금 보상만으로 계산한 손익분기는 1회당 평균 `313,500 CR`입니다. 보상 차량의 수집 가치와 남겨 둔 22B의 가치는 포함하지 않았습니다. 삭제한 22B의 가격은 회수되지 않습니다.

공식 Series 3 안내는 Auto Drive AFK XP·CR 파밍과 Estate 스킬 포인트 파밍 수정을 명시했습니다. 공식 문서에는 숫자가 없지만, 패치 직후 기존 EventLab 이벤트의 최종 유효 점수가 `50,000`점으로 제한되고 완료 1회당 최대 `1 SP`만 지급된다는 사용자 실측이 있습니다. 이 구 이벤트 경로는 v0.5.0 실행 경로에서 폐기했습니다.

새 경로는 `챌린지 플레이하기`의 코드 `122 697 651`입니다. 기본값은 `WaitTime=25`, `ResultWait=5`, `RunLoop=111`, `EventLabLoadWait=25`입니다. `결과 OCR 사용`은 기본 꺼짐이며 이때 성공으로 가정해 중간 회차는 Esc, 마지막 회차는 Enter를 사용합니다. 옵션을 켜면 실패 화면은 재시작 Enter·종료 Esc, 성공 화면은 재시작 Esc·계속 Enter로 판정합니다. 사용 전 1회 보상을 직접 확인하세요.

| 기본 111회 구성 | 시간/결과 |
| --- | ---: |
| 기본 주행 | `111 × 25초 = 2,775초` |
| 중간 재로딩 | `110 × 25초 = 2,750초` |
| 마지막 Enter 간격과 최종 로딩 | `5 + 25 = 30초` |
| 단독 스바루런 최소 합계 | 약 `5,555초` = `92분 35초` + 선택형 OCR 대기 |
| 관측 범위 기준 SP | `999~1,110 SP` (111회 × 관측 9~10 SP) |

최근 반복 확인에서는 회차당 10 SP가 거의 일관됐습니다. 기본값을 99회가 아닌 111회로 둔 이유는 스바루 프리셋 33대에 필요한 990 SP에 여유를 두기 위해서입니다. 과거 관측값 9 SP를 전 회차에 적용해도 `111 × 9 = 999 SP`가 되어 필요량을 넘고, 10 SP면 1,110 SP입니다. 올인원의 실제 SP 예산 검증을 최종 기준으로 사용합니다. 좋아요 확정 단계는 사용자 실검증이 끝난 안정화 동작입니다. 처음에는 `챌린지 1 / 구매 1 / 마스터리 1 / 삭제 0`으로 전체 흐름을 확인하세요.

- 공식 가격 조정: [Steam 398.92.0 패치노트](https://store.steampowered.com/news/app/2483190/view/689762798147733105)
- 공식 파밍 수정: [Forza Horizon 6 Series 3](https://forza.net/news/forza-horizon-6-series-3)
- 구 EventLab 이벤트 제한 실측: [EventLab skill points capped?](https://www.reddit.com/r/ForzaHorizon/comments/1uvjqp6/eventlab_skill_points_capped/)
- 최신 반복 확인은 기술 HUD 꺼짐에서 회차당 10 SP가 거의 일관됐고 60 FPS도 검증했습니다. 환경별 차이를 고려해 실제 SP 예산을 확인합니다.

## 4. 키와 중지

- `F1`: 현재 탭 실행
- `F2`: 중지 요청
- `F3`: 중지 요청 후 프로그램 종료
- `F4`: 휠스핀 진단 화면 저장

`F2`는 이미 게임에 전달된 입력을 취소하지 않습니다. 이상한 화면이 보이면 중지한 뒤 게임에서 현재 차량·메뉴·정렬을 직접 확인하세요.

실행 중에는 다른 키보드·마우스·게임패드 입력을 넣거나 다른 창으로 포커스를 옮기지 마세요. 포커스가 바뀌면 전역 키 입력이 다른 프로그램에 전달될 수 있습니다.

## 5. 작은 횟수로 확인

`테스트` 탭도 실제 구매·마스터리·삭제 입력을 보냅니다. 단순 미리보기가 아닙니다.

처음에는 다음처럼 확인합니다.

1. 삭제 횟수를 0으로 둡니다.
2. 런·구매·강화를 각각 1회 수준으로 줄입니다.
3. 프로그램 옆에서 모든 화면 전환을 관찰합니다.
4. 구매한 차량과 최근 추가됨 1번 칸을 사람이 확인합니다.
5. 삭제는 별도 탭에서 1회만 시험합니다.
6. 휠스핀 자동 판매는 가치가 낮은 환경에서 1회만 시험하거나 사용하지 않습니다.

계정 상태나 메뉴 위치가 바뀌면 이전 성공 기록을 그대로 재사용하지 마세요.

## 6. 탭별 시작 위치

### 스바루런

```text
챌린지 1회차 주행 화면
```

스바루 작업차 탑승, 코드 검색과 챌린지 진입은 사람이 미리 끝냅니다. F1을 누르면 다음 순서만 실행합니다.

1. 즉시 `W`를 누른 채 첫 회차를 25초 주행합니다.
2. 기본 설정은 성공 고정으로 중간 회차마다 `Esc`를 입력합니다. `결과 OCR 사용`을 켜면 같은 성공/실패 상태를 두 번 연속 확인한 뒤 해당 재시작 키를 입력합니다.
3. 마지막 회차는 확인된 상태의 종료 키를 입력하고 기본 5초 뒤 `Enter`로 좋아요를 확정합니다.
4. 좋아요 Enter 뒤 `W`를 떼고 오픈월드 로딩 25초를 기다립니다.

다른 차량으로 시작하면 안 됩니다. 기본 타이밍이 빠듯하므로 다른 키를 누르거나 게임 포커스를 옮기지 마세요.

### 구매

```text
자동차 컬렉션 > 제조사 > 구매할 차량 카드
```

자동차 매장·상품권·스토어는 시작 위치가 아닙니다. 단독 구매 탭은 현재 카드가 맞다고 가정하고 입력을 반복합니다.

올인원·테스트에서 사용하는 제조사·차량 이동 횟수는 계정의 DLC와 차량 목록에 따라 달라집니다. 게임 화면에서 직접 세어 입력하세요.

### 마스터리

```text
페스티벌 > 경매장 > 경매 시작 > 차량 선택 목록
```

프리셋은 구매·강화·삭제 횟수와 마스터리 입력 경로를 채웁니다. 선택된 차량이 실제 대상이라는 뜻은 아닙니다.

### 삭제

```text
페스티벌 > 경매장 > 경매 시작 > 차량 선택 목록
```

최근 추가됨 기준 첫 칸을 대상으로 삭제 입력을 반복합니다. 이미지 비교가 실행될 수 있지만, 그 결과만 믿고 대량 삭제하지 마세요.

작업 중 다른 차량을 받거나 휠스핀을 직접 열면 최근 추가됨 정렬이 바뀔 수 있습니다.

### 테스트와 올인원

```text
오픈월드
```

테스트·올인원은 스바루 작업차에 직접 탑승한 오픈월드에서 챌린지 코드를 검색한 뒤 주행·구매·마스터리·삭제를 연결합니다. 2회차부터는 삭제 후 오픈월드로 복귀해 `Esc → PageDown → Enter → Y → Enter → Esc → Backspace`로 즐겨찾기 차량의 제조사 목록을 열고, SUBARU 제조사와 1998 22B 작업차를 이미지로 다시 선택한 뒤 다음 챌린지에 들어갑니다. 중간 단계가 어긋나면 안전 정지할 수 있으므로 처음부터 큰 횟수로 실행하지 마세요.

각 챌린지 구간이 끝나면 구매 전에 `Esc → PageDown`으로 자동차 메뉴를 열어 현재 SP를 두 번 읽습니다. 필요량은 마즈다 `21 × 강화 대수`, 라부엘토 `39 × 강화 대수`, 레거시 스바루 `30 × 강화 대수`입니다. 16:9 FHD·QHD·4K 한국어 UI에서 두 판독값이 같고 현재 SP가 필요량 이상일 때만 계속합니다. 모든 OCR은 내부에서 FHD 좌표로 정규화하며 울트라와이드와 비 16:9 화면은 지원하지 않습니다.

### 일반·슈퍼 휠스핀

```text
나의 HORIZON > 실행할 일반/슈퍼 휠스핀 카드에 포커스
```

휠스핀은 테스트·올인원과 별도입니다. 목표 완료 후 오픈월드로 돌아가지 않고 휠 메뉴에 머무릅니다.

## 7. 휠스핀 처리

프로그램은 결과 색상, 차량 카드의 연식·제조사·일부 모델명, FE 표시 등을 추정해 중복차 팝업의 보관·판매 행을 선택합니다.

다음 상황에서는 판정이 틀릴 수 있습니다.

- OCR이 연식·제조사·모델을 잘못 읽음
- 독점 차량 목록이 게임 업데이트와 달라짐
- Forza Edition 표식이 가려지거나 다른 그래픽과 겹침
- 모션·플래시·팝업 타이밍이 기준보다 빠르거나 느림
- 게임 언어·해상도·색상 설정이 다름

가치가 큰 차량을 자동 판매하지 않으려면 보관 옵션을 넓히거나 자동 중복차 판매를 사용하지 마세요.

`상세 화면 저장`은 기본 꺼짐입니다. 오류·중지·F4 수동 캡처는 이 옵션과 관계없이 남을 수 있습니다.

보상 결과:

```text
result/
  latest-wheelspin-result.txt
  wheelspin-날짜-시간-종류/
    wheelspin-summary.rtf
    wheelspin-result.rtf
    wheelspin-result.txt
```

## 8. 자주 생기는 문제

### 메뉴 입력이 한 칸씩 밀림

- 해당 탭의 입력 간격을 늘립니다.
- 메뉴 진입 대기와 경매장 대기를 늘립니다.
- `다음에 할 일`이 꺼져 있는지 확인합니다.
- 시즌 전환이나 차량 추가 뒤 제조사·차량 이동 횟수를 다시 셉니다.

### 챌린지 검색이나 시작이 어긋남

- 16:9 화면·한국어 UI인지 확인하고 가능하면 FHD로 맞춥니다.
- 주행 보조 최대, 기술 HUD 꺼짐, 다음에 할 일 꺼짐인지 확인합니다.
- 오픈월드에서 시작했는지 확인합니다.
- 코드가 `122 697 651`인지 확인합니다.
- Xbox Game Pass·게임패드 가상 키보드나 텍스트 포커스가 남으면 현재 미지원 환경이므로 중지합니다.
- 로딩이 끝나기 전에 입력되면 `재로딩 대기`를 25초보다 늘립니다.

### 최근 추가됨 순서가 예상과 다름

자동화를 중지하고 직접 정렬을 확인하세요. 다른 신규 차량 수령, 직접 휠스핀, 구매 실패가 원인일 수 있습니다. 프로그램이 이전 단계의 구매 성공을 확정적으로 증명하지는 못합니다.

### 느린 PC에서 전환 전에 입력됨

로딩 대기, 메뉴 진입 대기, 구매 입력 간격, 경매장 대기를 늘립니다. 한 번에 하나만 바꾸고 횟수 1로 다시 관찰하세요.

### 설정을 초기화하고 싶음

1. 프로그램을 종료합니다.
2. `config/subaruu.conf`를 별도 위치에 보관합니다.
3. 원본을 제거하고 프로그램을 다시 실행합니다.

기본 설정 파일에서 새 사용자 설정이 만들어집니다.

## 9. 로그 확인과 공유

먼저 확인할 파일:

- `log/last-run-status.txt`
- 가장 최근 `log/session-*.log`
- 휠스핀이면 가장 최근 `wheelspin-session-*` 또는 `superwheel-session-*`

지원 요청에는 실패 직전·직후가 담긴 가장 작은 범위만 사용하세요. 로그의 Windows 사용자 이름과 절대 경로를 지우고, 화면 캡처의 게이머태그·CR·알림을 가립니다. `config/subaruu.conf` 전체를 보낼 필요가 없다면 관련 항목만 복사하세요.

프로그램은 이 자료를 자동 업로드하지 않습니다. 사용자가 메신저·이슈·이메일에 올리는 순간 해당 서비스의 개인정보 처리 조건이 적용됩니다.

## 10. v0.5.0 정식 범위

`v0.5.0`은 새 챌린지 경로, 선택형 결과 OCR, 올인원 회차 사이 작업차 재탑승과 16:9 OCR 정규화를 정식 기준으로 사용합니다. 구 EventLab 이벤트의 자동 진입 경로는 신 실행 흐름에서 호출하지 않습니다.

정식판이어도 화면 기반 입력이라는 한계는 같습니다. 처음 쓰는 PC, 게임 업데이트·시즌 전환 뒤, 설정을 바꾼 날에는 작은 횟수로 다시 확인하고 삭제·자동 판매는 사람이 대상을 확인한 뒤 사용하세요.
