# 보유 CR 적용 버튼과 안전 설명 2026-07-08

이 릴리즈는 보유 CR 입력칸 옆에 `적용` 버튼을 추가한 이전 빌드 기록입니다.
현재 배포는 이후 긴급 패치 릴리즈를 확인하세요.

## 다운로드

- Asset: `subaruu.zip`
- SHA256: `512969EE2B6190D39664DA9532FEC6D4470BC5D56074C68D21B6C1C677D4D964`

## 변경

- `올인원` 탭의 `보유CR` 옆에 `적용` 버튼을 추가했습니다.
- 오른쪽 `CR` 표시는 제거하고, 보유 CR 입력칸 바로 옆에 적용 버튼을 붙였습니다.
- `적용` 버튼을 누르면 현재 프리셋 차량 가격 기준으로 회차, 구매, 강화, 삭제 값이 바로 갱신됩니다.
- 보유 CR이 충분하면 값은 유지하고 예산 충분 로그를 남깁니다.
- 보유 CR이 부족하면 가능한 회차를 먼저 줄이고, 1회차 전체도 부족하면 가능한 구매 대수까지 줄입니다.
- 테스트 탭의 보유 CR 입력 요구를 제거했습니다.
- 테스트 탭의 수정 불가 `변경` 표시를 숨기고 맵 코드 입력칸을 정리했습니다.
- README와 GUIDE에 보유 CR 계산 방식, 적용 버튼, 부족할 때의 자동 제한 규칙을 자세히 적었습니다.

## 보유 CR 계산 기준

- 계산식: `차량 가격 x 구매 대수 x 회차 = 총 필요 CR`
- 스바루작: `81,700 CR`
- 라부엘토작: `346,750 CR`
- 마즈다작: `95,000 CR`
- 입력값은 숫자만 사용하며 최대 `999999999`까지 허용합니다.

## 확인

- `dotnet build -c Release`
- `dotnet publish -c Release -r win-x64 --self-contained true`
- ZIP 안쪽 루트 폴더가 `subaruu/` 하나인 것 확인
- ZIP 안에 `subaruu.conf`, `subaruu.conf.bak`, `log` 폴더가 포함되지 않는 것 확인
- 패키지 SHA256 계산 완료

## 문제 보고

문제가 생기면 `F2`로 즉시 멈추고, `subaruu.exe` 옆의 `log` 폴더와 `subaruu.conf`를 같이 보내주세요.
`last-run-status.txt`도 `log` 폴더 안에 함께 저장됩니다.
