# v0.5.0-rc.4

## 반드시 확인

- 첫 회차는 반드시 `1998 SUBARU IMPREZA 22B-STI VERSION` 작업차를 탄 상태에서 시작하세요.
- 테스트·올인원의 기본 챌린지 코드는 `122 697 651`입니다.
- 새 맵은 성공을 전제로 하므로 `결과 OCR 사용`은 기본 꺼짐입니다. 기본 주행시간은 25초입니다.
- 장시간 실행과 자리를 비운 무인 이용은 권장하지 않습니다. 먼저 챌린지 1회와 올인원 2회차를 직접 확인하세요.
- 4K 144Hz·기술 표시 꺼짐 조건에서 5회 50 SP를 확인했습니다. 이전 9 SP 관측도 있어 실제 SP 예산 검증을 최종 기준으로 사용합니다.

## 변경 사항

- 테스트와 올인원의 기본 맵 코드를 `122 697 651`로 통일했습니다.
- 결과 OCR을 체크박스로 선택할 수 있게 했습니다.
  - 기본 꺼짐: 성공으로 가정, 중간 회차 `Esc`, 마지막 회차 `Enter`
  - 켜짐: 성공·실패 결과를 OCR로 판정해 재시작·종료 키 선택
- 기본 주행시간을 25초로 변경했습니다.
- 휠스핀 탭의 기본 종류를 일반에서 슈퍼로 변경했습니다.
- 모든 OCR 입력을 16:9 FHD 좌표로 정규화했습니다. FHD·QHD·4K를 지원하며 울트라와이드와 비 16:9 화면은 지원하지 않습니다.
- 슈퍼 휠스핀에서 중복차 팝업이 빈 화면 없이 다음 차량으로 바로 바뀌는 경우도 새 팝업으로 처리합니다.
- 마스터리 SP 숫자가 정확한 숫자 토큰으로 읽힌 경우의 판정 기준을 보강했습니다.
- 테스트·올인원 다회차의 2회차 이후 스바루 작업차 자동 재선택을 유지합니다.
- 마지막 좋아요 Enter는 사용자 실게임 검증이 끝난 안정화 동작으로 유지합니다.
- `1989 NISSAN S-CARGO`와 `2020 WULING` 오인 경우를 모두 FE 후보로 보호합니다. FE 표식 판독이 불확실해도 판매하지 않습니다.

## 알려진 문제

Xbox Game Pass 또는 게임패드 중심 환경에서는 공유 코드 입력 뒤 가상 키보드나 텍스트 입력 포커스가 남아 검색으로 진행되지 않을 수 있습니다. 물리 키보드 Enter도 반응하지 않고 마우스 클릭 뒤에만 진행되는 사례가 있습니다. 이 환경의 코드 입력 호환 처리는 아직 확정 해결이 아니므로, 코드 입력 화면에서 멈추면 `F2`로 중지하고 수동으로 복구하세요.

이 문제의 후속 설계는 `GAMEPASS_CODE_INPUT_DESIGN.md`에 기록했습니다.
