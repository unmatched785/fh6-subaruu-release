# Xbox Game Pass·게임패드 공유 코드 입력 호환 설계

> v0.5.0에서는 이 환경을 현재 지원 범위 밖의 알려진 한계로 둡니다. 아래 내용은 정식판의 필수 승격 조건이 아니라, 향후 충분한 실화면 자료가 확보됐을 때 검토할 후속 설계입니다.

## 문제 정의

일부 Xbox Game Pass 또는 게임패드 중심 환경에서는 챌린지 공유 코드 숫자가 입력된 뒤에도 검색 단계로 넘어가지 않습니다. 물리 키보드의 Enter를 수동으로 눌러도 반응하지 않고 게임 화면을 마우스로 클릭한 뒤에야 진행되는 사례가 보고됐습니다.

다음 제보는 같은 원인으로 취급합니다.

- 테스트 모드가 공유 코드 입력 후 멈춤
- 가상 키보드 때문에 올인원이 진행되지 않음
- 수동 Enter도 먹지 않고 마우스 클릭 뒤에만 진행됨

핵심 원인 가설은 숫자 입력 실패가 아니라 가상 키보드 또는 텍스트 입력 위젯이 포커스를 계속 소유하는 것입니다. 따라서 Enter를 반복 전송하는 것만으로는 해결되지 않습니다.

## 설계 원칙

1. 가상 키보드를 강제로 비활성화하는 것을 기본 해결책으로 삼지 않습니다. Windows·Steam·Game Pass·게임패드 설정에 따라 동작이 다릅니다.
2. 현재의 키보드 입력 경로는 정상 작동하는 사용자를 위해 유지합니다.
3. 진행 화면을 확인하지 않은 상태에서 임의 좌표를 클릭하지 않습니다.
4. 마우스 호환 처리는 전체 화면 절대 좌표가 아니라 게임 클라이언트 영역과 16:9 정규화 좌표를 사용합니다.
5. 검색 결과 화면이 확인되지 않으면 다음 Enter 연타나 맵 선택을 하지 않고 안전 정지합니다.

## 제안 상태 머신

1. `OpenCodeField`
   - 공유 코드 입력창을 엽니다.
   - 입력창 또는 가상 키보드 표시 여부를 화면 캡처로 기록합니다.
2. `EnterDigits`
   - 현재 숫자 직접 입력 방식을 사용합니다.
   - 가능하면 입력된 9자리 문자열을 선택·복사하여 검증합니다.
3. `KeyboardSubmit`
   - 기존 방식으로 Enter를 한 번 전송합니다.
   - 짧은 시간 동안 검색 결과 화면 전환을 이미지 신호로 확인합니다.
4. `PointerFocusFallback`
   - 전환이 없고 Game Pass/게임패드 호환 옵션이 활성화된 경우에만 실행합니다.
   - 확보된 화면 자료에서 검증된 안전 영역 또는 검색 버튼을 이미지로 찾습니다.
   - 해당 지점을 한 번 클릭하여 텍스트/가상 키보드 포커스를 해제합니다.
5. `PointerSubmitFallback`
   - 포커스 해제만으로 검색이 실행되지 않으면 검증된 검색/확정 버튼을 한 번 클릭합니다.
6. `VerifySearchResults`
   - 검색 결과 카드 또는 결과 목록의 고유 화면 특징을 두 프레임 연속 확인합니다.
   - 확인된 경우에만 아래 방향키와 맵 선택 Enter 단계로 넘어갑니다.
7. `FailSafeStop`
   - 어느 단계에서도 결과 화면을 확인하지 못하면 캡처와 단계명을 저장하고 긴급정지합니다.

## 향후 UI 제안

- 설정 이름: `Game Pass/게임패드 코드 입력 호환`
- 값: `자동`, `키보드만`, `마우스 보강`
- 권장 기본값: `자동`
- `자동`은 키보드 제출을 먼저 시도하고 화면 전환이 없을 때만 마우스 보강을 사용합니다.

## 구현 전에 필요한 자료

영향받는 환경에서 다음 FHD 캡처 또는 원본 해상도 캡처가 필요합니다.

1. 코드 입력창을 열기 전
2. 가상 키보드가 열린 상태
3. 9자리 코드 입력 완료 상태
4. Enter가 먹지 않고 멈춘 상태
5. 마우스 클릭 직후 진행된 상태

가능하면 같은 실행의 세션 로그와 클릭한 실제 위치도 함께 보존합니다. QHD·4K는 FHD 기준 위치가 확정된 뒤 16:9 정규화로 검증합니다.

## 완료 기준

- 키보드 사용 환경의 기존 동작을 깨뜨리지 않습니다.
- Game Pass/게임패드 환경에서 코드 입력 후 검색 결과까지 자동 진행합니다.
- 테스트와 올인원 모두 같은 상태 머신을 사용합니다.
- 잘못된 맵 결과나 미전환 화면에서 맵 선택 Enter를 보내지 않습니다.
- 실패 시 `code-input`, `keyboard-submit`, `pointer-fallback`, `search-result-verify` 중 정확한 중단 단계와 캡처가 남습니다.

## v0.6.0-rc.2 백그라운드 실험과의 관계

백그라운드 실험 모드는 이 문서의 화면 판정·포인터 보강을 구현한 것이 아닙니다. OCR과 화면 캡처 없이 코드 숫자와 검색 Enter를 FH6 창에 직접 게시하는 별도 RC 경로입니다.

- Steam/일반 키보드 경로부터 검증합니다.
- Xbox Game Pass 가상 키보드가 열린 상태에서 동작한다고 가정하지 않습니다.
- 코드 입력 성공이나 검색 결과를 화면으로 확인하지 않습니다.
- 테스트 회차 1, 구매·강화·삭제 0으로 제한해 실패 뒤 위험 단계로 진행하지 않습니다.
- 실게임 검증에 실패하면 고정 좌표 클릭이나 전경 탈취를 자동 대안으로 추가하지 않습니다.
